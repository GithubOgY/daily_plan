# 社会福祉法人もるどう会 あじさい園 ブログ記事企画書

## 1. 企画の背景・目的

### 背景
- 社会福祉法人もるどう会 あじさい園として、地域での認知度向上と情報発信が重要である
- 既存のブログを活用し、より効果的な情報発信を目指す
- 利用者・家族、地域住民とのコミュニケーションツールとしてブログを活用
- 誠実で信頼できる情報発信を通じて、事業所の存在や活動を広く知ってもらう

### 目的
1. **事業所の認知度向上・情報発信**: 事業所の活動や理念を多くの方に知っていただく
2. **利用者・家族への情報提供**: 利用者やそのご家族に役立つ情報を提供し、安心して利用していただく
3. **地域住民とのコミュニケーション**: 地域の方々との関係構築と相互理解の促進
4. **誠実な情報発信**: 正確で信頼できる情報を発信し、事業所の信頼性を高める
5. **読者に楽しんでもらう**: 堅苦しくなく、読者が楽しみながら読める内容を心がける

### ブログの位置づけ
- **バズを狙わない**: 過度に派手な表現や炎上を狙うような内容は避ける
- **継続的な情報発信**: 週1回のペースで継続的に情報を発信
- **信頼性の構築**: 誠実で正確な情報発信により、長期的な信頼関係を構築

## 2. ターゲット読者設定

### プライマリターゲット

#### 利用者・家族
- **既存利用者**: 事業所の活動や情報を知りたい方
- **利用者のご家族**: 事業所での活動内容や支援内容を理解したい方
- **利用を検討している方とその家族**: 事業所の雰囲気や活動を知りたい方
- **年齢層**: 20代〜60代
- **関心事**: 就労支援、日常生活の支援、事業所での活動内容

#### 地域住民
- **近隣住民**: 事業所の存在や活動を知りたい方
- **地域の企業・団体**: 協働・連携の可能性を探っている方
- **年齢層**: 30代〜70代
- **関心事**: 地域の福祉活動、社会貢献、地域コミュニティ

### セカンダリターゲット

#### 福祉関係者
- **他事業所の職員**: 情報交換やネットワーキング
- **行政関係者**: 事業所の取り組みを理解したい方
- **専門家**: 福祉分野の専門家や研究者

#### 一般の方々
- **福祉に関心のある方**: 社会福祉や障害者支援に関心を持つ方
- **ボランティア希望者**: ボランティア活動に関心のある方

## 3. ブログのコンセプト・トーン

### コンセプト
**「誠実で親しみやすい、あじさい園の日常と想いを伝えるブログ」**

- **誠実さ**: 正確で信頼できる情報を提供
- **親しみやすさ**: 堅苦しくなく、読みやすい文章
- **温かみ**: 事業所の温かい雰囲気が伝わる内容
- **楽しさ**: 読者が楽しみながら読める工夫

### トーン&マナー
- **親しみやすい**: 「です・ます」調を基本とし、堅苦しくない表現
- **誠実**: 誇張や過度な表現を避け、事実に基づいた内容
- **前向き**: 明るく前向きなトーンを保つ
- **分かりやすさ**: 専門用語には補足説明を加える
- **温かみ**: 利用者や職員への思いやりが感じられる表現

### 避けるべき表現
- 過度に派手な表現や煽り文句
- 誇張や事実と異なる内容
- 個人を特定できる情報の公開
- 不確かな情報や噂

## 4. 記事カテゴリと企画案

**3. 季節の活動**

- 夏: 夏祭り、暑さ対策、夏の装飾
- 秋: 作品展、収穫祭、文化祭
- 冬: クリスマス装飾、年末年始の活動

**4. 施設紹介**
- 事業所の各スペースの紹介
- 作業スペース、休憩スペース、食堂など

- 安全で快適な環境づくり

**5. 年間行事**

- 利用者の皆さんの楽しそうな様子

**6. 日常の小さな出来事**
- 利用者の皆さんの成長の瞬間
- 職員と利用者の交流
- 日々の小さな発見や喜び
- 温かいエピソード

## 5. 記事の構成テンプレート

### 基本構成

#### 1. 導入部（リード文）
- **目的**: 読者の興味を引く、記事の内容を簡潔に伝える
- **文字数**: 100-200文字程度
- **内容**: 
  - 記事のテーマを提示
  - 読者が読みたくなるような導入
  - 簡潔で分かりやすい文章

#### 2. 本文
- **目的**: 記事の本題を詳しく説明
- **構成**: 
  - 見出し（H2、H3）を使って構造化
  - 段落を適切に分ける
  - 箇条書きや表を活用
  - 写真や図を適切に配置

#### 3. まとめ
- **目的**: 記事の要点を整理し、読者に伝えたいことを明確に
- **文字数**: 100-200文字程度
- **内容**: 
  - 記事の要点を簡潔にまとめる
  - 読者へのメッセージ
  - 今後の予告（あれば）

#### 4. 関連リンク・お問い合わせ
- **目的**: 読者が次のアクションを起こしやすくする
- **内容**: 
  - 関連記事へのリンク
  - お問い合わせ先の案内
  - 見学・体験の案内
  - SNSへのリンク

### 記事構成の例

```
【タイトル】

【導入部】
（記事のテーマを簡潔に紹介。読者の興味を引く内容）

【本文】

（本文の内容）

### 見出し2
（詳細な説明）

- 箇条書き1
- 箇条書き2
- 箇条書き3

（写真や図を配置）

## 見出し1（続き）
（本文の続き）

（記事の要点をまとめる。読者へのメッセージ）

【関連リンク・お問い合わせ】
- お問い合わせ: [連絡先]
- 見学・体験のご案内: [案内ページへのリンク]
```

## 6. 執筆のポイント

### 文章の書き方

#### 基本原則
- **「です・ます」調**: 親しみやすく、読みやすい
- **簡潔に**: 長い文章は避け、短い文章を心がける
- **分かりやすく**: 専門用語には補足説明を
- **具体的に**: 抽象的な表現より具体的な表現を

#### 読者が楽しめる工夫
- **エピソードを交える**: 具体的なエピソードで親近感を持ってもらう
- **写真を活用**: 視覚的に分かりやすく、楽しめる
- **季節感を取り入れる**: 季節に合わせた内容で親しみやすく
- **読者を意識する**: 読者が知りたいことを意識して書く

#### 誠実さを保つ工夫
- **事実に基づく**: 事実を正確に伝える
- **誇張を避ける**: 過度な表現は使わない
- **不確かな情報は書かない**: 確認できない情報は載せない
- **個人情報に配慮**: 利用者や職員の個人情報を守る

### 見出しの付け方
- **階層構造**: H2、H3を適切に使い、構造化する
- **分かりやすく**: 見出しだけで内容が分かるように
- **興味を引く**: 読者が読みたくなる見出しを

### 写真の活用
- **適切な枚数**: 1記事に3-5枚程度を目安
- **個人情報に配慮**: 利用者の顔が特定されないよう配慮
- **説明文を付ける**: 写真の説明（キャプション）を付ける
- **質の良い写真**: 明るく、見やすい写真を選ぶ

## 9. 執筆ガイドライン

### 文字数の目安
- **短い記事**: 800-1,200文字（日常の小さな出来事、簡単な報告など）
- **標準的な記事**: 1,500-2,500文字（通常の記事）
- **長い記事**: 3,000文字以上（制度解説、詳細な報告など）
- **目安**: 読者が5-10分で読める長さ

### 見出しの付け方
- **H1**: 記事タイトル（1記事に1つ）
- **H2**: 大見出し（記事の主要なセクション）
- **H3**: 小見出し（H2の下の詳細なセクション）
- **見出しの数**: 1記事にH2を3-5個程度

### 段落の分け方
- **1段落**: 1つのテーマについて3-5文程度
- **改行**: 適切に改行し、読みやすく
- **空行**: 段落の間に空行を入れる

### 写真の活用
- **枚数**: 1記事に3-5枚程度
- **配置**: 本文の適切な位置に配置
- **説明**: 写真の説明（キャプション）を付ける
- **個人情報**: 利用者の顔が特定されないよう配慮

### SEO対策（控えめに）
- **キーワード**: 自然な形でキーワードを含める
- **タイトル**: 検索されやすいタイトルを心がける
- **見出し**: 見出しにもキーワードを含める（自然に）
- **過度な対策は避ける**: 自然な文章を優先

### チェックリスト
記事を公開する前に以下を確認：
- [ ] 誤字・脱字がないか
- [ ] 個人情報が適切に保護されているか
- [ ] 事実に基づいた内容か
- [ ] 読者に分かりやすい文章か
- [ ] 写真の説明が適切か
- [ ] 関連リンクが正しく設定されているか

## 10. 成果測定

### 測定指標

#### アクセス数
- **ページビュー数**: 記事がどれだけ読まれているか
- **ユニークユーザー数**: どれだけの人が訪問しているか
- **滞在時間**: 読者がどれだけの時間記事を読んでいるか
- **離脱率**: 読者が途中で離れていないか

#### 読者の反応
- **コメント数**: 読者からのコメント
- **シェア数**: SNSなどでのシェア
- **いいね数**: 記事への反応
- **問い合わせ数**: 記事を読んでの問い合わせ

#### 問い合わせ数
- **見学・体験の問い合わせ**: 記事を読んでの問い合わせ
- **利用相談の問い合わせ**: 制度に関する問い合わせ
- **採用に関する問い合わせ**: 職員募集に関する問い合わせ

### 月次レビュー
- **月1回**: 投稿データを分析
- **人気の記事**: どの記事が読まれているか
- **改善点**: 改善できる点を見つける
- **次月の計画**: 分析結果を次月の計画に反映

### 目標設定（例）
- **3ヶ月後**: 月間アクセス数 500PV
- **6ヶ月後**: 月間アクセス数 1,000PV、問い合わせ数 月2-3件
- **1年後**: 月間アクセス数 2,000PV以上、定期的な問い合わせ

## 11. リスク管理

### 個人情報保護
- **利用者の個人情報**: 顔写真、氏名、住所などは一切公開しない
- **職員の個人情報**: 職員の許可を得てから公開、必要に応じて匿名化
- **家族の情報**: 家族の情報も保護
- **撮影許可**: 写真を撮影する際は必ず許可を得る

### 情報の正確性
- **事実確認**: 記事の内容は事実に基づいているか確認
- **制度情報**: 制度に関する情報は最新のものを確認
- **不確かな情報**: 確認できない情報は載せない
- **訂正**: 誤りがあった場合は速やかに訂正

### コメント管理
- **コメント確認**: コメントは公開前に確認
- **不適切なコメント**: 不適切なコメントは削除
- **返信**: 読者からの質問には丁寧に返信
- **プライバシー**: コメントに個人情報が含まれていないか確認

### 著作権・肖像権
- **写真の使用権限**: 写真の使用権限を確認
- **利用者の作品**: 利用者の作品を紹介する際も許可を得る
- **他者の著作物**: 他者の著作物を使用する際は許可を得る

## 12. 運用体制

### 担当者
- **メイン担当**: ブログ担当職員
- **サブ担当**: 他の職員1-2名（バックアップ体制）
- **承認者**: 管理者（必要に応じて）

### 業務分担
- **記事執筆**: メイン担当が中心に実施
- **写真撮影**: 職員全員で協力
- **校正・チェック**: メイン・サブ担当が分担
- **公開**: メイン担当が実施（必要に応じて承認を得る）
- **コメント対応**: メイン・サブ担当が分担

### 業務時間
- **1記事あたり**: 執筆2-3時間、写真撮影・編集1時間、合計3-4時間程度
- **週1回の投稿**: 週に1回、3-4時間程度の時間を確保
- **効率化**: テンプレートを活用し、効率的に作業

### 承認フロー
1. **執筆**: メイン担当が記事を執筆
2. **校正**: サブ担当が校正・チェック
3. **承認**: 必要に応じて管理者が承認
4. **公開**: メイン担当が公開

### 連携
- **SNSとの連携**: InstagramやX（Twitter）でブログ記事を紹介
- **Webサイトとの連携**: Webサイトからブログへのリンク
- **他媒体との連携**: チラシやパンフレットでブログを紹介

## 13. まとめ

この企画書を基に、社会福祉法人もるどう会 あじさい園のブログ記事を作成・公開していきます。重要なポイントは以下の通りです：

1. **誠実な情報発信**: 正確で信頼できる情報を提供する
2. **読者に楽しんでもらう**: 堅苦しくなく、読みやすい内容を心がける
3. **継続性**: 週1回のペースで継続的に情報を発信する
4. **個人情報保護**: 利用者や職員のプライバシーを最優先に
5. **バランス**: 4つのカテゴリをバランスよく配置する
6. **地域とのつながり**: 地域の方々との関係構築を重視する

ブログは、事業所の日常や活動、想いを伝える大切なツールです。誠実で親しみやすい情報発信を通じて、多くの方々に事業所の存在や理念を知っていただき、より良い地域社会づくりにつなげていきましょう。

### 次のステップ

1. **記事の執筆**: この企画書のテーマに基づいて記事を執筆
2. **写真の準備**: 記事に使用する写真を準備
3. **校正・チェック**: 記事の内容を確認
4. **公開**: ブログに記事を公開
5. **SNSでの紹介**: InstagramやX（Twitter）で記事を紹介
6. **振り返り**: 月1回、投稿データを分析し、改善点を見つける
